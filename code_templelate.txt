(수정) 24.05.06. 소스코드명 소문자시작 -> 대문자 시작

명칭에 관한 규칙
- 상수의 이름은 대문자와 언더스코어(_)로만 작성한다.
  ex) POS_SIZE = 4; | COL_SIZE = 100;

- 변수명은 소문자로 시작하고, 두번째 단어는 대문자로 시작한다.
  ex) int sumValue; | int colNumber;

- 함수명은 소문자로 시작하며, 첫 번째 단어는 동사로 작성한다. 두번째 단어는 대문자로 시작한다.
  ex) int getNumber(); | void movePosition(); 

- 소스코드명은 대문자로 시작하고, 두번째 단어는 대문자로 시작한다.
  ex) SnakeMove.cpp 

- 포인터명은 p로 시작하며, 참조하는 변수명의 첫 글자를 대문자로 구성하여 작성한다.
 

주석에 관한 규칙
- 코드의 첫 주석에는 최초 작성일, 해당 소스파일 작성자, 목적, 개정 이력을 주석으로 기재한다.
ex)
// [24.04.29.] 이재원, snake의 움직임을 저장하고 나타낸다. 
// [24.04.30.] 이재원, getSnakePos() 발생 가능 예외 추가 및 예외 처리 완료

- 함수 정의 앞에는 리턴값, 함수의 용도, 매개변수, 발생가능예외, 변경 이력을 주석으로 기재한다.
ex)
// 리턴값 : int / 매개변수 : 없음
//	snake의 position을 반환하는 함수
// 발생 가능 예외: 
//	snake의 움직일 수 있는 범위를 벗어난 position 반환 시 예외 발생 가능
int getSnakePos()

- 함수 인자에 대한 설명, 복잡한 논리식, 간단하지 않은 자료구조에 관한 내용에는 필히 주석을 작성한다. 


문장에 관한 규칙
- switch 문에서 case 문을 빠져나오기 위해 break 문을 사용한다. 
  (break문을 사용하지 않고, 다음 case로 넘어갈것을 의도하는 경우, 주석을 작성하여 이를 표시한다.)
- switch 문에서 case 문이 다 끝난 부분에 default 문을 넣는다.
- break 문은 가능한한 한 번만 사용한다.
- if-else 문의 끝은 else로 마무리짓는다.
